# Security Policy

## Supported Versions
Security updates are applied to `main`.

| Version | Supported |
| --- | --- |
| main | Yes |
| other branches | No |

## Reporting a Vulnerability
Use GitHub Security Advisories when available.
If unavailable, open an issue with prefix `[SECURITY]` and avoid exploit details.

Include:
- Affected model, schema, or script
- Impact and severity
- Reproduction steps

## Response Targets
- Initial triage: up to 72 hours
- Mitigation plan: up to 7 business days

## Scope
This policy covers SQL schema, Node/Sequelize models, tests, and CI workflows.
